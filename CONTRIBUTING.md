# Contributing to Planify

Thank you for your interest in contributing to Planify! This document provides guidelines and instructions for contributing.

## Getting Started

1. **Fork the repository** and clone it locally
2. **Install dependencies** required for the project
3. **Create a branch** for your contribution

## Project Structure

- **Frontend** - Input collection form and results display interface
- **Backend API** - Intent normalization, destination filtering, and scoring logic
- **Core Logic** - Recommendation engine and itinerary generation module
- **Data Layer** - Destination data, activities, and metadata

## How to Contribute

### Reporting Bugs

- Check existing issues to avoid duplicates
- Use a clear, descriptive title
- Include steps to reproduce the issue
- Provide environment details (OS, browser, etc.)

### Suggesting Features

- Open an issue with the `enhancement` label
- Describe the feature and its use case
- Explain how it fits into the system flow

### Submitting Code

1. Create a feature branch from `main`
   ```bash
   git checkout -b feature/your-feature-name
   ```
2. Make your changes following the code style guidelines
3. Write or update tests as needed
4. Commit with clear, descriptive messages
5. Push to your fork and open a Pull Request

## Code Style Guidelines

- Use consistent indentation and formatting
- Write meaningful variable and function names
- Add comments for complex logic
- Keep functions focused and modular

## Pull Request Process

1. Ensure your code passes all tests
2. Update documentation if needed
3. Link related issues in your PR description
4. Request review from maintainers
5. Address feedback promptly

## Areas for Contribution

- **Intent Normalization** - Improve parsing of user inputs
- **Destination Filtering** - Add new constraint types
- **Scoring Algorithm** - Enhance ranking factors
- **Itinerary Generation** - Optimize day-wise planning
- **Weather Integration** - Expand external API support
- **Documentation** - Improve guides and examples

## Code of Conduct

- Be respectful and inclusive
- Provide constructive feedback
- Help others learn and grow

## Questions?

Open an issue with the `question` label or reach out to the maintainers.

---

Thank you for helping make Planify better!
